# Информация о проекте Diplom3D

**Diplom3D** — это веб-приложение для автоматического построения 3D-моделей этажей зданий на основе фотографий планов эвакуации. Проект объединяет обработку изображений, построение векторной модели, генерацию 3D, навигацию по помещениям, редактирование результатов и объединение нескольких планов в один.

Проще говоря, система принимает изображение плана, обрабатывает его, выделяет стены, комнаты и двери, строит 3D-модель этажа, а затем позволяет просматривать её, редактировать и строить маршрут между помещениями.

---

## Что уже реализовано

### 1. Загрузка и хранение данных

В системе уже реализована загрузка планов эвакуации и вспомогательных файлов. После загрузки файл сохраняется, а информация о нём записывается в базу данных. Для этого используются отдельные сущности для загруженных файлов и реконструкций.

Также реализовано хранение результатов обработки:
- исходные изображения;
- маски;
- промежуточные данные векторизации;
- итоговые 3D-модели;
- информация о комнатах, зданиях и этажах.

Это позволяет не пересчитывать всё заново при каждом открытии проекта.

---

### 2. Обработка изображений планов

В проекте уже есть полноценный pipeline обработки изображений. Он включает:
- предобработку изображения;
- бинаризацию;
- удаление цветных элементов и текста;
- поиск контуров;
- распознавание комнат и дверей;
- определение масштаба;
- нормализацию координат.

Также система умеет распознавать текст на плане с помощью OCR и использовать его для привязки номеров помещений.

Иными словами, проект уже не просто показывает картинку, а извлекает из неё полезную геометрию и данные.

---

### 3. Построение векторной модели

После обработки изображения система строит векторное представление плана. В нём стены, комнаты и двери уже представлены не как пиксели, а как геометрические объекты.

Это важно, потому что векторная модель используется дальше:
- для построения 3D;
- для редактирования;
- для навигации;
- для stitching нескольких планов.

Сейчас это один из ключевых результатов работы backend.

---

### 4. Построение 3D-модели

На основе векторной модели и маски система строит 3D-модель этажа.

Уже реализовано:
- преобразование 2D-геометрии в 3D-сцену;
- extrude стен;
- экспорт моделей в форматы OBJ и GLB;
- загрузка и просмотр 3D-модели во frontend.

Пользователь может открыть готовую модель в браузере и визуально оценить результат обработки.

---

### 5. Построение маршрутов

В проекте уже есть навигационная часть. Она строится на основе графа, в котором вершины соответствуют проходимым точкам, а рёбра — возможным переходам между ними.

Используется алгоритм A* для поиска кратчайшего пути.

Сейчас реализованы:
- построение навигационного графа на уровне processing;
- поиск маршрута между комнатами;
- отображение маршрута во frontend;
- UI для выбора начальной и конечной точки.

Это даёт возможность показывать путь между помещениями поверх 3D-модели.

---

### 6. Редактирование плана

Пользователь может не только смотреть результат, но и редактировать его.

Поддерживаются сценарии:
- изменение кропа изображения;
- поворот плана;
- редактирование маски;
- добавление или корректировка комнат и дверей;
- повторное построение навигационного графа;
- пересборка 3D-модели.

Также есть отдельный экран для ручной правки плана и редактор маски.

---

### 7. Stitching нескольких планов

В проекте реализован отдельный сценарий stitching — объединение нескольких планов в один.

Система уже умеет:
- выбрать несколько реконструкций;
- применить transform и crop;
- использовать clipping polygons;
- объединить результаты;
- нормализовать итоговую модель;
- сохранить stitched reconstruction.

Для этого есть и backend-логика, и отдельный интерфейс во frontend.

---

### 8. Авторизация и администрирование

В системе уже есть JWT-аутентификация и отдельные страницы для авторизации.

Также реализована административная часть:
- список реконструкций;
- просмотр сохранённых проектов;
- удаление реконструкций;
- список ожидающих пользователей;
- admin layout.

Это позволяет использовать систему не только как демонстрационный инструмент, но и как полноценное приложение с разделением ролей.

## Какие технологии использовались

### Backend

- **Python 3.12** — основной язык backend;
- **FastAPI** — HTTP API;
- **SQLAlchemy** — работа с базой данных;
- **Alembic** — миграции;
- **OpenCV** — обработка изображений;
- **NumPy** — работа с массивами;
- **scikit-image** — дополнительные операции с изображениями;
- **pytesseract** — OCR;
- **shapely** — геометрические операции;
- **trimesh** — построение и экспорт 3D-моделей;
- **NetworkX** — графы и навигация;
- **JWT** — аутентификация.

### Frontend

- **React 18** — интерфейс;
- **TypeScript** — типизация;
- **Vite** — сборка;
- **React Router** — маршрутизация;
- **Axios** — запросы к backend;
- **Three.js** и **@react-three/fiber** — 3D-визуализация;
- **Fabric.js** — редактирование плана и маски;
- **Zustand** — управление состоянием;
- **Lucide React** — иконки.

### Хранение и инфраструктура

- **SQLite** — используется в разработке;
- **PostgreSQL** — используется в production;
- **pytest** — тестирование backend;
- **ESLint** — проверка frontend;
- **Git** — версионирование.

## Как устроен проект

### Backend

Backend отвечает за всю прикладную логику:
- загрузку файлов;
- обработку изображений;
- создание масок;
- векторизацию;
- построение 3D;
- навигацию;
- stitching;
- сохранение результатов в базе данных.

По сути, это ядро системы.

### Frontend

Frontend отвечает за взаимодействие с пользователем:
- загрузку изображения;
- пошаговую обработку;
- редактирование результата;
- просмотр 3D-модели;
- построение маршрута;
- stitching;
- административные экраны.

### База данных

База данных хранит:
- пользователей;
- загруженные файлы;
- реконструкции;
- комнаты;
- здания и этажи;
- связанные параметры обработки;
- ссылки на готовые 3D-файлы.

## Что работает, а что ещё нет

### Работает

- загрузка файлов;
- обработка изображений;
- OCR и удаление текста;
- выделение контуров;
- построение векторной модели;
- построение 3D-модели;
- просмотр модели в браузере;
- построение маршрутов;
- редактирование плана;
- stitching;
- авторизация;
- список реконструкций;
- удаление реконструкций.

### Частично реализовано

- часть навигационного HTTP-контракта ещё возвращает заглушки;
- отдельные ручки для комнат и обновления реконструкции не завершены;
- часть сценариев использует временное определение пользователя;
- некоторые данные в frontend ещё приводятся через fallback-логику.

### Технический долг

- не везде соблюдено полное разделение слоёв;
- часть логики всё ещё живёт в API-роутерах;
- некоторые сервисы слишком большие и отвечают сразу за несколько задач;
- часть данных хранится в JSON-поле, а не полностью в нормализованной структуре;
- frontend местами использует касты и дублирование загрузки данных;
- некоторые элементы UI пока реализованы как временные или неполные решения.

## Почему проект важен

Проект показывает полный путь от фотографии плана эвакуации до интерактивной 3D-модели здания. Это полезно для задач:
- визуализации объектов;
- навигации внутри зданий;
- подготовки цифровых моделей помещений;
- анализа и редактирования планов;
- построения маршрутов эвакуации.

То есть система объединяет компьютерное зрение, 3D-графику, навигационные алгоритмы и веб-интерфейс в одном приложении.

## Короткий итог для выступления

Diplom3D — это уже работающая система, которая:
- принимает план эвакуации;
- обрабатывает изображение;
- строит векторную модель;
- создаёт 3D-модель этажа;
- даёт редактировать результат;
- строит маршрут между помещениями;
- поддерживает stitching нескольких планов;
- хранит результаты и даёт их просматривать через веб-интерфейс.

При этом проект ещё развивается: часть API и внутренних слоёв требует доработки, а в кодовой базе остаётся технический долг, который планируется закрывать поэтапно.
