## Business Context

Конечный пользователь приходит на публичную главную ([PublicHomePage](frontend/src/pages/PublicHomePage.tsx)), вводит «ДВФУ» в поиск, кликает на подсказку — и должен попасть на 3D-просмотр этажей с возможностью построить маршрут между помещениями. Это headline-сценарий ВКР: «зашёл — увидел корпус в 3D — нашёл аудиторию».

Текущее состояние: страница [FloorViewerPage](frontend/src/pages/FloorViewerPage.tsx) и её хук [useFloorViewer](frontend/src/hooks/useFloorViewer.ts) уже почти полностью реализуют целевой UI (header с хлебными крошками, селекторы корпус/этаж/отсек, минимапа отсеков, 3D-вьюер, поля построения маршрута). Но клик на «ДВФУ» в [PublicHomePage.tsx:117](frontend/src/pages/PublicHomePage.tsx:117) ведёт на несуществующий `/map`, а загрузка каталога ломается на 401: `GET /buildings?published=true` ([buildings_hierarchy.py:42-55](backend/app/api/buildings_hierarchy.py:42)) защищена `Depends(security)`, хотя по смыслу это публичный режим.

Фича — **bug-fix / surface-polish**: убрать три блокера, проверить наличие данных и улучшить UX поля «Начальная точка/Конечная точка» (сейчас принимает только строку формата `"D304"`).

## Acceptance Criteria

1. **Доступность из публичной главной.** Неавторизованный пользователь, открывший `/`, кликнув на «ДВФУ» в выпадашке, попадает на `/viewer`, видит каталог зданий и 3D-сцену первого опубликованного отсека ДВФУ без 401 и без редиректа на `/login`.
2. **Каталог без авторизации.** `GET /api/v1/buildings?published=true` отдаёт `200` без `Authorization`-заголовка; `GET /api/v1/buildings` (admin-режим, `published=false`) продолжает требовать токен.
3. **Маршрут работает анонимно.** `POST /api/v1/navigation/multifloor-route` строит маршрут для неавторизованного пользователя (уже публичен — проверить регрессионно).
4. **Корректная навигация.** На странице переключение `Корпус → Этаж → Отсек` через карусели и клик по минимапе меняет загружаемый GLB; селекторы синхронизированы.
5. **Понятный ввод комнат.** Поля «Начальная точка» / «Конечная точка» предлагают автокомплит из списка известных комнат текущего здания вместо свободного ввода кода вида `"D304"`. (Если объём слишком велик — оформляется в отдельном тикете, минимально: подсказка с примером формата над полем.)
6. **Данные ДВФУ присутствуют.** В БД есть `Building` с кодом, попадающим под опубликованные критерии, и хотя бы один отсек с готовой реконструкцией (`mesh_url_glb != null`). Если данных нет — задокументирован seed-скрипт или шаги ручного посева.
7. **Регрессии нет.** Существующие админские страницы (`/admin/buildings`, `/admin/floor-editor`, `/admin/route-test`) продолжают работать.
