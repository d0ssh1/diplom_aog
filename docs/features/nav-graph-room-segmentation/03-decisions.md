# Design Decisions: nav-graph-room-segmentation

## Decisions

| # | Decision | Choice | Alternatives | Rationale |
|---|----------|--------|--------------|-----------|
| 1 | Алгоритм выделения коридоров | `distanceTransform` свободного пространства + порог | Дилатация стен (текущий), watershed, ML-сегментация | distanceTransform напрямую измеряет ширину прохода в пикселях — это именно то, что нужно. Дилатация косвенная и масштабо-зависимая. Watershed и ML избыточны для данной задачи. |
| 2 | Источник порога | `k * wall_thickness_px`, где `wall_thickness_px` уже вычислен в `nav_service.py:56` | Фиксированный порог, порог из конфига | `wall_thickness_px` уже есть в пайплайне (`pipeline.py:569`). Адаптивный порог устраняет зависимость от масштаба плана. |
| 3 | Значение коэффициента k | `k = 1.5` (default) | k = 1.0, k = 2.0 | Коридор должен быть шире стены. k=1.5 означает: проход шире 1.5 толщины стены = коридор. Параметр вынесен в аргумент функции для тестируемости. |
| 4 | Минимальный порог | `max(3.0, k * wall_thickness_px)` | Без минимума | При `wall_thickness_px = 0` (нет стен) или очень тонких стенах порог не должен быть нулевым — иначе весь план станет коридором. |
| 5 | Обратная совместимость сигнатуры | Добавить `wall_thickness_px: float` как обязательный параметр, убрать `dilate_kernel_size` / `dilate_iterations` | Оставить старые параметры как deprecated | Старые параметры больше не нужны — новый алгоритм их не использует. Единственный вызов в `nav_service.py:59` — обновляется одновременно. |
| 6 | Финальная дилатация для восстановления границ | Сохранить: `dilate(corridor_rough) AND free_space` | Убрать | Дилатация после порога нужна для заполнения «дырок» в маске коридора (distanceTransform даёт только центральные пиксели широких проходов). Размер ядра = `int(wall_thickness_px)` вместо фиксированного 7. |

## Risks

| Risk | Impact | Mitigation |
|------|--------|-----------|
| `wall_thickness_px = 0` при пустой маске | High — порог = 0, весь план = коридор | Зажать порог снизу: `max(3.0, k * wall_thickness_px)` |
| Планы без явных коридоров (open-plan) | Med — крупнейший компонент = вся площадь | Ограничение `max_corridor_area = 0.5 * total` сохраняется из текущей реализации |
| Регрессия на планах, где текущий алгоритм работал | Med | Тесты с синтетическими планами разных масштабов |

## Open Questions

- [x] Нужно ли сохранять параметры `dilate_kernel_size` / `dilate_iterations` для обратной совместимости? — **Нет.** Единственный вызов в `nav_service.py:59`, обновляется одновременно.
- [x] Где вычислять `wall_thickness_px` — в `nav_graph.py` или передавать снаружи? — **Передавать снаружи** через `nav_service.py`. `processing/` не должен импортировать из `pipeline.py` (правило архитектуры).
- [ ] Нужен ли параметр `k` в API (чтобы пользователь мог настраивать)? — Пока нет. Вынесен в аргумент функции для тестируемости, но не экспонируется в API.
