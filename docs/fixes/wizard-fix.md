## Проблемы

**1. Phase A4 — автоповорот может зацикливаться**

В StepPreprocess `useEffect` проверяет `naturalHeight > naturalWidth` и вызывает `onRotate()`. Но после поворота на 90° компонент перерендерится, `useEffect` сработает снова, увидит что теперь `naturalWidth > naturalHeight` — ок, но если `rotation` уже 90° и изображение по-прежнему вертикальное в DOM (CSS transform не меняет `naturalWidth/Height`), то проблемы не будет. Однако стоит добавить guard: вызывать `onRotate` только если `rotation === 0`. Иначе при повторном монтировании может повернуть дважды.

**Рекомендация:** Добавить в phase-A4 уточнение:
```
Auto-rotate ТОЛЬКО если rotation === 0 (первый раз). 
Не поворачивать повторно при перемонтировании.
```

**2. Phase B1 — стена рисуется на mouse:up, но должна на второй клик**

В описании написано "click-click (две точки → прямая линия)", но реализация описана через `mouse:down / mouse:move / mouse:up` — это drag, не два отдельных клика. Claude Code может реализовать как drag (зажал → протянул → отпустил = линия), а не как клик-клик.

**Рекомендация:** Уточнить в phase-B1:
```
ВАЖНО: Стена рисуется ДВУМЯ ОТДЕЛЬНЫМИ КЛИКАМИ, не drag!
- Первый mouse:down (без move) → установить startPoint, показать preview
- mouse:move (без зажатой кнопки) → обновлять preview линию  
- Второй mouse:down → установить endPoint, создать финальную линию, сбросить startPoint
- НЕ использовать mouse:up для завершения — это сломает UX
```

**3. Phase B1 — Shift-snap описан, но нет деталей реализации**

Claude Code может пропустить или реализовать криво.

**Рекомендация:** Добавить конкретный код:
```typescript
// Shift snap: привязка к ближайшей оси
if (e.e.shiftKey) {
  const dx = Math.abs(endX - startX);
  const dy = Math.abs(endY - startY);
  if (dx > dy) { endY = startY; }  // горизонталь
  else { endX = startX; }           // вертикаль
}
```

**4. Phase B3 — maskUrl формируется неточно**

Написано: `maskUrl: /api/v1/uploads/${state.maskFileId}`. Но в реальном `reconstruction_service.py` маски сохраняются в `uploads/masks/{maskFileId}.*`. URL на фронте должен быть `/api/v1/uploads/masks/{maskFileId}.png` (или как настроен static serving).

**Рекомендация:** Проверить текущий паттерн URL маски в существующем коде (StepEditMask или apiService) и использовать тот же.

**5. Phase B1 — getBlob() экспортирует canvas включая аннотации**

`canvas.toDataURL()` экспортирует ВСЁ на canvas: и маску, и оранжевые прямоугольники кабинетов, и зелёные линии дверей. Но бэкенду нужна чистая маска (белые стены на чёрном фоне), без цветных аннотаций.

**Рекомендация:** Добавить в phase-B1:
```
getBlob() должен экспортировать ТОЛЬКО маску (стены + ластик), 
БЕЗ аннотаций (rooms, doors). Перед экспортом:
1. Скрыть все annotation-объекты (group.visible = false)
2. canvas.toDataURL()
3. Восстановить видимость (group.visible = true)

Все annotation-объекты должны иметь custom property data.type = 'annotation'
для фильтрации.
```

**6. Нет описания что происходит при "Назад" на шаге 3**

Если пользователь нажмёт "Назад" на шаге 3 (редактор стен), он вернётся на шаг 2 (препроцессинг). Все нарисованные стены и аннотации потеряются. Это может быть неожиданно.

**Рекомендация:** Либо подтверждение "Вы уверены? Изменения будут потеряны", либо сохранять canvas state в wizard state.
