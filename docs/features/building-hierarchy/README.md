# Building Hierarchy — Design

date: 2026-05-05
status: draft

## Business Context

Сейчас один этаж в системе соответствует одному плану/реконструкции. В реальности этаж корпуса состоит из нескольких отсеков (см. макет ДВФУ: корпус D, этаж 7, отсеки 2..9), каждый из которых сфотографирован отдельно и проходит через визард как самостоятельный план. Без понятия «отсек» невозможно собрать целостный этаж: 3D-модели висят без географической привязки, end-user UI не может показать «карту этажа», навигация через границы отсеков невозможна.

Цель фичи — ввести строгую иерархию **Building → Floor → Section → Reconstruction**, дать админу инструмент для разметки схемы этажа (где какой отсек) и привязки висящих планов к отсекам, а пользователю — селектор «Корпус → Отсек → Этаж» с подсветкой активного отсека на мини-карте этажа (как на референс-макете).

Лестницы и лифты не реализуются в этой фиче, но `Section.section_type` оставляет место для будущих типов отсеков-переходов.

## Acceptance Criteria

1. **Иерархия в БД:** `Reconstruction.floor_id` (FK → floors), `Section.floor_id` (FK → floors), `Section.reconstruction_id` (FK → reconstructions, UNIQUE, nullable), `Floor.building_id` (FK → buildings, активная связь), `Building.code` (короткий буквенный код корпуса, UNIQUE)
2. **Старые реконструкции снесены** миграцией: дроп таблицы reconstructions+rooms+floor_transitions, пересоздание с FK
3. **Wizard:** на шаге «Загрузка» (StepUpload) поля `building` и `floor` обязательны для перехода дальше — выбор из существующих корпусов и этажей; сохранение реконструкции (PUT `/save`) отправляет `floor_id`, `Section.reconstruction_id` остаётся NULL
4. **CRUD корпусов и этажей:** API `/buildings`, `/buildings/{id}/floors` для админа; экран `/admin/buildings` для управления (create/edit/delete с подтверждением)
5. **Редактор отсеков** (`/admin/floor-editor`, label «Редактор отсеков»):
   - **Multi-step wizard** (шаги 1→2→3→4→5) для нового/незавершённого этажа:
     - Шаг 1 — загрузка фото-схемы этажа (JPG/PNG/PDF), сохраняется в `Floor.schema_image_id`
     - Шаг 2 — кадрирование + поворот (orange handles), сохраняется в `Floor.schema_crop_bbox`
     - Шаг 3 — извлечение стен: автоматически (CV pipeline) + ручная корректировка инструментами «Выделение стен» / «Прямоугольник» / «Очистить всё», сохраняется в `Floor.wall_polygons`
     - Шаг 4 — разметка отсеков прямоугольником (повёрнутый rect = 4-точечный полигон), модалка `NewSectionDialog` (поверх canvas) — ввод номера
     - Шаг 5 — привязка отсеков к планам через галерею карточек: поиск + dropdown'ы Здание/Этаж (default «Все/Все»), допускаются планы любого этажа
   - **Overview mode** (если этаж уже размечен) — графический вид + контекстное меню («Изменить номер», «Удалить отсек»); кнопка «Сохранить изменения»
   - **Table mode** — табличный вид (Номер / План / Статус / Действия), кнопка «Редактировать схему» возвращает в Wizard step 1
6. **EditPlanPage:** в правом нижнем углу плашка «Привязан к отсеку №N» / «Не привязан» с ссылкой на редактор схемы этажа
7. **End-user UI** (новый `FloorViewerPage`):
   - три селектора: Корпус (с буквами `code`), Отсек, Этаж — с подсветкой активного
   - мини-карта этажа: фото-схема (cropped+rotated по `Floor.schema_crop_bbox`) как фон + `Floor.wall_polygons` чёрными линиями + полигоны секций; **активная секция — оранжевая заливка**
   - клик на отсек в селекторе или на мини-карте → грузится 3D-mesh реконструкции, привязанной к этому отсеку
   - корпуса без хотя бы одного полностью размеченного этажа не показываются
8. **Совместимость с FloorTransition:** существующая фича маршрутов через несколько отсеков продолжает работать на `MultifloorRouteResponse` — backend-логика маршрутизации не меняется
9. **Тесты:** новые ORM, repos, service, API endpoints покрыты по правилам [prompts/testing.md](../../../prompts/testing.md)

## Out of Scope

- Лестницы и лифты как отдельный тип отсека (только зарезервировано поле `section_type`)
- Автоматическая детекция отсеков с фото-схемы (отсеки рисуются вручную)
- Поля `Section.description`, `Section.color` (фиксированный orange = active, neutral = idle)
- Экспорт схемы (на мокапе шаг 8 — out of scope)
- Background-task обработка wall extraction (sync вызов в первой версии)
- Многокорпусные комплексы внутри одного Building
- Версионирование схемы этажа (перепланировки во времени)
- Совместная работа нескольких админов над схемой одного этажа
