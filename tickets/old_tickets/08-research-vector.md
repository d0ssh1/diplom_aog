## Проблема

Текущий пайплайн маски теряет тонкие линии стен и скругляет углы. Как бы пользователь ни крутил ползунки Чувствительность/Контраст — часть чёрных линий с оригинального плана не попадает в маску.

### Что теряется:
- Тонкие внутренние перегородки (1–2px на скане)
- Дверные проёмы и их обводки
- Тонкие линии нижнего ряда стен
- Прямые углы стен (скругляются)
- Пунктирные линии (маршруты эвакуации) — частично

### Текущий пайплайн:
```
Фото → rotate → remove_colored_elements (зелёные стрелки, красные символы)
→ crop → grayscale → adaptive threshold (blockSize, thresholdC)
→ MORPH_CLOSE (3,3) iter=1 → text removal → PNG
```

### Почему adaptive threshold не справляется:

Adaptive threshold (`cv2.adaptiveThreshold`) вычисляет порог **локально** для каждого пикселя на основе среднего в окне `blockSize × blockSize`. Параметр `C` — смещение от этого среднего.

**Конфликт параметров:**
- Маленький `blockSize` (7–11) + маленький `C` (2–5) → захватывает тонкие линии, но добавляет шум (зернистость фона)
- Большой `blockSize` (25–51) + большой `C` (10–20) → чистый фон, но тонкие линии теряются
- **Нет одного набора параметров, который захватит всё**

**Дополнительные потери:**
- `MORPH_CLOSE (3,3)` скругляет прямые углы и «склеивает» близкие линии
- Фото плана неравномерно освещено → разные области требуют разных параметров

## Фаза 1: Research

Исследовать и протестировать следующие подходы. Для каждого — замерить качество на 3–5 тестовых планах.

### R1. Multi-pass adaptive threshold (комбинирование)

**Идея:** запустить adaptive threshold несколько раз с разными параметрами и объединить результаты через `bitwise_OR`.

```python
# Проход 1: крупные стены (текущие параметры)
mask1 = cv2.adaptiveThreshold(gray, 255, cv2.ADAPTIVE_THRESH_MEAN_C,
                               cv2.THRESH_BINARY_INV, blockSize=15, C=10)

# Проход 2: тонкие линии (мелкое окно, малый C)
mask2 = cv2.adaptiveThreshold(gray, 255, cv2.ADAPTIVE_THRESH_MEAN_C,
                               cv2.THRESH_BINARY_INV, blockSize=7, C=3)

# Проход 3: средние элементы
mask3 = cv2.adaptiveThreshold(gray, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
                               cv2.THRESH_BINARY_INV, blockSize=11, C=6)

# Объединение
combined = cv2.bitwise_or(mask1, cv2.bitwise_or(mask2, mask3))
```

**Плюсы:** просто, параметризуемо, сохраняет текущую архитектуру  
**Минусы:** может добавить шум от прохода с мелким окном  
**Сложность:** Низкая (30 мин)

### R2. CLAHE preprocessing (выравнивание контраста перед бинаризацией)

**Идея:** применить Contrast Limited Adaptive Histogram Equalization до threshold — это выравнивает неравномерную освещённость фото.

```python
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))
enhanced = clahe.apply(gray)
# Потом обычный adaptive threshold
```

**Плюсы:** решает проблему неравномерного освещения, тонкие линии становятся контрастнее  
**Минусы:** может усилить шум текстуры бумаги  
**Сложность:** Низкая (15 мин)

### R3. Canny edge detection + threshold fusion

**Идея:** Canny находит все края (включая тонкие линии), adaptive threshold находит заполненные области. Объединить:

```python
edges = cv2.Canny(gray, 50, 150)
# Утолщить edges чтобы были видны как стены
edges_dilated = cv2.dilate(edges, np.ones((2, 2)), iterations=1)

threshold_mask = cv2.adaptiveThreshold(...)

combined = cv2.bitwise_or(threshold_mask, edges_dilated)
```

**Плюсы:** Canny отлично ловит тонкие линии и сохраняет углы  
**Минусы:** Canny ловит ВСЕ края, включая текст, стрелки, мусор  
**Сложность:** Средняя (нужна фильтрация мусора после Canny)

### R4. Морфологическое восстановление углов

**Идея:** заменить `MORPH_CLOSE` на более аккуратную операцию, которая не скругляет углы:

```python
# Вместо MORPH_CLOSE с квадратным ядром:
# Два прохода с линейными ядрами (горизонтальный + вертикальный)
kernel_h = np.ones((1, 3), np.uint8)
kernel_v = np.ones((3, 1), np.uint8)
closed_h = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel_h)
closed_v = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel_v)
result = cv2.bitwise_or(closed_h, closed_v)
```

**Плюсы:** закрывает разрывы в линиях без скругления углов  
**Минусы:** не решает проблему потери тонких линий  
**Сложность:** Низкая (15 мин)

### R5. Sauvola / Niblack binarization (альтернатива adaptive threshold)

**Идея:** алгоритмы Sauvola и Niblack — более продвинутые локальные методы бинаризации, специально разработанные для документов.

```python
from skimage.filters import threshold_sauvola

thresh = threshold_sauvola(gray, window_size=15, k=0.2)
binary = (gray < thresh).astype(np.uint8) * 255
```

**Плюсы:** Sauvola лучше справляется с неравномерным фоном, разработан для документов  
**Минусы:** требует `scikit-image`, может быть медленнее  
**Сложность:** Низкая (20 мин)

### R6. Hough Line Transform (детекция прямых линий)

**Идея:** после базовой бинаризации — найти прямые линии через Hough и «дорисовать» пропущенные:

```python
edges = cv2.Canny(gray, 50, 150)
lines = cv2.HoughLinesP(edges, 1, np.pi/180, threshold=50,
                         minLineLength=30, maxLineGap=10)
# Нарисовать найденные линии на маске
for line in lines:
    x1, y1, x2, y2 = line[0]
    cv2.line(mask, (x1, y1), (x2, y2), 255, 2)
```

**Плюсы:** находит даже еле заметные прямые линии, сохраняет прямые углы  
**Минусы:** не найдёт кривые элементы, может «додумать» линии где их нет  
**Сложность:** Средняя

## Фаза 2: Design

На основе результатов Research — спроектировать улучшенный пайплайн.

### Ожидаемая структура (гипотеза):

```
Фото → rotate → remove_colored_elements → crop
→ CLAHE (выравнивание контраста)                     [R2]
→ Multi-pass adaptive threshold (3 прохода, OR)       [R1]
→ Canny edges + фильтрация → OR с threshold           [R3]
→ Directional MORPH_CLOSE (H + V ядра, OR)            [R4]
→ text removal → PNG
```

- Тонкие внутренние перегородки (1–2px на скане) видны на маске

- Дверные проёмы и обводки захватываются
- Фон остаётся чистым (нет зернистости)
- Время генерации маски <2 секунд
