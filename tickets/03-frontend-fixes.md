## Fix 1: Публичная стартовая страница (НОВЫЙ ЭКРАН)

**Проблема:** Приложение начинается сразу с Dashboard/Login. Должна быть публичная стартовая страница для обычных пользователей (не админов).

**Макет:** `docs/design/10_public_home.png`

**Что сделать:**

Создать `pages/PublicHomePage.tsx` — стартовый экран для всех:

Layout:
- **Header:** оранжевая полоска слева (~100px) + белый фон. Слева: "SUPER_DIPLOM" (чёрный bold). Справа: кнопка "Войти как администратор" → navigate('/login')
- **Основная область:** белый фон, по центру:
  - Заголовок "Введите название или код" (чёрный, bold, ~32px)
  - Поисковый инпут с иконкой лупы справа (чёрная рамка, ~500px ширина)
  - Это поиск по зданиям/помещениям для обычных пользователей

**Роутинг — обновить `App.tsx`:**
```tsx
<Route path="/" element={<PublicHomePage />} />      {/* публичная страница */}
<Route path="/login" element={<LoginPage />} />       {/* вход админа */}
<Route path="/admin" element={<AppLayout />}>          {/* админка */}
  <Route index element={<DashboardPage />} />
  <Route path="mesh/:id" element={<ViewMeshPage />} />
</Route>
<Route path="/upload" element={<WizardPage />} />
```

После успешного логина — redirect на `/admin`, не на `/`.

## Fix 2: Wizard Step 1 — Загрузка файлов

**Проблема:** Реализация не соответствует макету. DropZone слишком большая, нет горизонтальной ленты миниатюр внизу, MetadataForm не как в макете.

**Макет:** `docs/design/08_upload_wireframe.png` (вайрфрейм), `docs/design/04_upload_single_metadata.png` (финальный)

**Что исправить в `StepUpload.tsx`:**

Layout должен быть таким:
```
┌──────────────────────────────────────────────────────────┐
│  Header + Step Indicator                                  │
├────────────────────────┬─────────────────────────────────┤
│                        │  Название картинки         [×]  │
│   DropZone             │  ┌─────────────────────────┐    │
│   (компактная,         │  │                         │    │
│    по центру)          │  │   Превью изображения    │    │
│                        │  │                         │    │
│   ┌─────────┐          │  └─────────────────────────┘    │
│   │ иконка  │          │  ← стрелки навигации →          │
│   │ облако  │          │                                  │
│   │         │          │  Здание    > ___________        │
│   └─────────┘          │  Этаж      > ___________        │
│   Перетащите           │  Крыло     > ___________        │
│   для загрузки         │  Блок      > ___________        │
│                        │                                  │

│ └──┘ └──┘ └──┘ └──┘   │                                  │
```

Ключевые отличия от текущей реализации:
1. **DropZone компактнее** — не на всю левую половину, а по центру с padding
2. **Горизонтальная лента миниатюр** внизу левой панели (если загружено >1 файла) — маленькие превью с × для удаления и именем файла
3. **Правая панель при загрузке 1 файла:** тёмно-серый фон, сверху — заголовок с именем файла + оранжевый ×, большое превью, стрелки ← → для навигации между файлами, внизу — MetadataForm (Здание, Этаж, Крыло, Блок)
4. **MetadataForm:** лейблы BOLD слева, инпуты с placeholder "> Здание" серый текст, нижняя граница (underline input style, не box)
5. **Поддержка множественной загрузки:** `<input type="file" multiple>` и drag-drop нескольких файлов одновременно

**Стиль инпутов MetadataForm:**

## Fix 3: Wizard Step 2 — Редактор маски (КРИТИЧНО)

**Проблема:** Текущая реализация ужасная — MaskEditor и ToolPanel не интегрированы, панель инструментов сломана, canvas маленький.

**Макеты:** `docs/design/07_edit_mask.png` (Figma), `docs/design/09_crop_wireframe.png` (кадрирование)

**Что исправить в `StepEditMask.tsx`:**

Layout:
```
┌──────────────────────────────────────────────────────────┐
│  Header + Step Indicator (шаг 2 активен — оранжевый)     │
├──────────────────────────────────────┬───────────────────┤
│                                      │ // Кадрирование   │
│                                      │ ┌────┐  ┌────┐   │
│                                      │ │crop│  │auto│   │
│     Canvas с изображением            │ └────┘  └────┘   │
│     (MaskEditor — Fabric.js)         │                   │
│                                      │ // Редактировать  │
│     Занимает ~75% ширины             │ ┌────┐  ┌────┐   │
│     Высота: весь viewport            │ │draw│  │erase│  │
│                                      │ └────┘  └────┘   │
│                                      │                   │
│                                      │ // Толщина        │
│                                      │ ●━━━━━━━━  6 px  │
│                                      │                   │
├──────────────────────────────────────┴───────────────────┤
│  [Назад]                                    [> Далее]    │
└──────────────────────────────────────────────────────────┘
```

Требования:
1. **Canvas (MaskEditor) занимает ~75% ширины и всю высоту** между header и footer. Изображение масштабируется с сохранением пропорций (object-fit: contain).
2. **ToolPanel справа (~25%, ~280px)** — тёмно-серый фон #4A4A4A
3. **Иконки-кнопки 80×80px** с закруглёнными углами 12px:
   - Активный инструмент: оранжевый фон (#FF5722)
   - Неактивный: серый фон (#9E9E9E)
4. **Секции** с заголовками "// Кадрирование", "// Редактировать", "// Толщина" — italic bold, белый текст
5. **Слайдер толщины:** белая полоса, чёрный ползунок, значение "6 px" справа
6. **Автоповорот:** если изображение вертикальное (height > width), автоматически повернуть на 90° для горизонтального отображения
7. **Интеграция с MaskEditor:** ToolPanel передаёт `activeTool` и `brushSize` в MaskEditor через props/callbacks. При смене инструмента в ToolPanel — MaskEditor переключает режим.

**Автоповорот — добавить в `useWizard.ts` или `StepEditMask.tsx`:**
```typescript
// При загрузке изображения проверяем ориентацию
const img = new Image();
img.onload = () => {
  if (img.height > img.width) {
    // Вертикальное изображение — повернуть на 90°
    setRotation(90);
  }
};
img.src = planUrl;
```

## Fix 4: Кадрирование (отдельный подшаг в Step 2)

**Макет:** `docs/design/09_crop_wireframe.png`

**Что сделать:**

Когда пользователь нажимает кнопку Crop в ToolPanel:
- Canvas переключается в режим кадрирования (CropSelector)
- Справа от canvas — MetadataForm (Этаж, Здание, итд)
- Внизу canvas — две кнопки: "Кадрировать" (применить) и "Авто. кадрирование"
- "Авто. кадрирование" вызывает backend API для auto-crop
- После кадрирования — вернуться в режим рисования

## Fix 5: Множественная загрузка файлов

**Проблема:** Текущая реализация поддерживает только 1 файл.

**Макет:** `docs/design/05_upload_multiple.png`

**Что исправить:**

1. В `useFileUpload.ts`: `addFile` → `addFiles(files: File[])`, поддержка массива
2. В `DropZone.tsx`: `<input type="file" multiple accept="image/*">`, `onDrop` обрабатывает `e.dataTransfer.files` (все файлы)
3. В `StepUpload.tsx`:
   - Если загружено >1 файла — правая панель показывает FileGrid (сетка 3×N с оранжевыми × для удаления)
   - Если загружен 1 файл — правая панель показывает превью + MetadataForm
   - Навигация стрелками ← → между файлами (меняет активный файл в превью)
