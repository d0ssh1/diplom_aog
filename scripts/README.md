# scripts/

Утилитарные скрипты для разработки и проверки данных.

## create_superuser.py

Создаёт суперпользователя (администратора) в базе данных. Нужен при первом
запуске проекта — после `alembic upgrade head` база пустая, войти некому.

**Запуск из корня проекта:**

```bash
cd backend
python ../scripts/create_superuser.py
```

По умолчанию создаёт пользователя `admin` с паролем `admin`.
Для кастомных учётных данных:

```bash
python ../scripts/create_superuser.py --username myname --password secret123
```

Если пользователь с таким именем уже существует — скрипт повышает его до
суперпользователя (не падает с ошибкой).

**Почему нельзя просто зарегистрироваться через форму?**
Регистрация через `/api/v1/users/` создаёт пользователя с `is_active=False`
(ожидает одобрения администратором). Но если администратора нет — круговая
зависимость. Этот скрипт разрывает её.

## check_dvfu_published.py

Sanity-проверка БД: есть ли в системе хотя бы один опубликованный корпус
(здание с секцией, у которой `reconstruction.status == Done`), и в частности
хотя бы один корпус ДВФУ. Используется перед демо ВКР, чтобы убедиться, что
страница `/viewer` не окажется пустой.

**Запуск из корня проекта:**

```bash
python scripts/check_dvfu_published.py
```

**Exit codes:**

- `0` — хотя бы один корпус ДВФУ опубликован
- `1` — публикаций ДВФУ нет (или вообще нет опубликованных корпусов)

Если данных нет — скрипт печатает чек-лист ручного посева через админ-UI.
