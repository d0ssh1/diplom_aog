## Проблема

Линия навигационного маршрута в 3D-сцене (шаг 5 wizard'а) **смещена относительно стен** — координаты маршрута в 2.5 раза больше, чем координаты меша.

### Корневая причина

В тикете 22 `scale_factor` был исправлен с `0.05` на `0.02` в `nav_service.py → build_graph()`, чтобы соответствовать `mesh_builder.py` (`pixels_per_meter=50`, `scale=1/50=0.02`). Однако:

1. **Старый `_nav.json`** на диске всё ещё содержит `"scale_factor": 0.05` (был сгенерирован до тикета 22).
2. **Фоллбэк в `find_route`** использует старое значение по умолчанию:
   ```python
   # nav_service.py, метод find_route(), строка ~98
   scale_factor = metadata.get('scale_factor', 0.05)  # ← БАГ: должно быть 0.02
   ```

### Числовой пример (proof)

Для узла комнаты в пиксельных координатах `(589, 141)` на маске `863×368`:

| | mesh_builder (правильно) | nav route (текущее, сломано) |
|---|---|---|
| **scale** | 0.02 | 0.05 |
| **x_3d** | 589 × 0.02 = **11.78** | 589 × 0.05 = **29.45** |
| **z_3d** | (141 − 368) × 0.02 = **−4.54** | (141 − 368) × 0.05 = **−11.35** |
| **Смещение** | — | **×2.5 от правильных координат** |

### Формулы трансформации (для справки — уже совпадают)

**mesh_builder.py** (через `mesh_generator.py → contours_to_polygons` + rotation −π/2 по X):
```
x_3d = x_pix / pixels_per_meter  = x_pix × 0.02
z_3d = (y_pix - H) / pixels_per_meter = (y_pix - H) × 0.02
```

**nav_graph.py → transform_2d_to_3d** (строки 529-550):
```python
x_3d = x_pix * scale_factor
z_3d = (y_pix - mask_height) * scale_factor
y_3d = y_offset  # 0.1 — чтобы линия левитировала над полом
```

Формулы **идентичны** при `scale_factor = 0.02`. Проблема только в значении scale_factor, которое берётся из JSON.

## Контекст: почему mesh не центрирует по X

`mesh_builder.py` → `contours_to_polygons()` делает:
```python
x = x_pix / pixels_per_meter          # НЕ (x_pix - W/2)
y = (H - y_pix) / pixels_per_meter    # Y-flip
```
Затем rotation −π/2 по X переводит Y→Z: `z = -y = -(H - y_pix)/ppm = (y_pix - H)/ppm`

Итого:
```
x_3d = x_pix * 0.02
z_3d = (y_pix - H) * 0.02
```

`transform_2d_to_3d` делает ровно то же самое. **Формулы совпадают.** Единственная проблема — значение scale_factor из JSON.
