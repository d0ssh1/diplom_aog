# Room Labels 3D — Design

date: 2026-05-25
status: draft

## Business Context

Сейчас пользователь, просматривая готовую 3D-модель здания в `WizardPage`
или `EditPlanPage`, видит только серые стены без каких-либо подписей. Он не может
понять, где находится аудитория 301, где коридор, где лестница — все выглядит
одинаково.

При построении маршрута (RouteTestPage / StepView3D) система уже умеет
отображать 2 комнаты (начальную и конечную) как полупрозрачные 3D-блоки
с подписями над ними (компонент `RoomMarker` в `FloorRouteView.tsx`). Пользователь
видит ровно тот стиль, который нужно распространить на ВСЕ комнаты при обычном
просмотре.

Данные для этого уже есть: `state.rooms: RoomAnnotation[]` хранится в wizard,
`GET /reconstructions/{id}/vectors` возвращает `VectorizationResult.rooms`
с нормализованными центрами и полигонами.

## Acceptance Criteria

1. В `MeshViewer` есть кнопка-тоггл «Кабинеты» (по умолчанию выкл.).
2. При включении тоггла все комнаты отображаются как полупрозрачные 3D-блоки
   с подписью над каждым — стиль аналогичен `RoomMarker` из route building.
3. Цвет блока соответствует типу комнаты (classroom=жёлтый, corridor=синий и т.д.).
4. Подписи корректно перемещаются при вращении камеры (Html из drei).
5. При выключении тоггла подписи и блоки исчезают.
6. Работает в двух контекстах: `StepView3D` (wizard) и `EditPlanPage` (сохранённая реконструкция).
7. Нет утечек памяти: Three.js-геометрии удаляются при unmount.
